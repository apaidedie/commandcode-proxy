================================================================================
  🚀 三种部署方式
================================================================================

方式 1：一键部署（最简单）
  $ chmod +x deploy.sh
  $ ./deploy.sh

方式 2：Makefile（推荐）
  $ make deploy
  $ make logs
  $ make health

方式 3：Docker Compose（标准）
  $ cp .env.example .env
  $ docker compose up -d
  $ docker compose logs -f

================================================================================
  🔧 配置说明
================================================================================

环境变量（.env）：
  PROXY_PORT=3050                          # 宿主机端口
  CC_API_BASE=https://api.commandcode.ai   # API 地址
  LOG_LEVEL=info                           # 日志级别
  TZ=Asia/Shanghai                         # 时区

资源限制：
  CPU: 1 核心（限制）/ 0.25 核心（预留）
  内存: 512MB（限制）/ 128MB（预留）

日志配置：
  单文件最大: 10MB
  保留文件数: 3 个
  驱动: json-file

================================================================================
  ✅ 快速验证
================================================================================

1. 健康检查：
   $ curl http://localhost:3050/health
   # 应返回: OK

2. 模型列表：
   $ curl http://localhost:3050/v1/models \
     -H "Authorization: Bearer user_YOUR_API_KEY"

3. 运行测试：
   $ chmod +x test.sh
   $ API_KEY=user_YOUR_API_KEY ./test.sh
